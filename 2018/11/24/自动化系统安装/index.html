PXE+DHCP自动安装

DHCP：(Dynamic Host Configuration Protocol)

yum install dhcp

DHCP的工作流程：

1. 客户端发起一个DHCP DISCOVER请求发现当前局域网中DHCP的服务器
2. DHCP服务器收到客户端请求后，响应DHCP OFFER，确定一个没有使用的IP地址给客户端
3. 客户端收到后，发送DHCP REQUEST确定使用该地址
4. 最后DHCP服务器发送DHCP ACK确认客户端的连接后，发送DHCP服务器的相关配置给客户端

DHCP服务的原则：

1. 本地优先
2. 先到先得

配置DHCP sever
默认配置文件：/etc/dhcp/dhcpd.conf
/etc/dhcp/dhcpd.conf，默认为空，使用/usr/share/doc/dhcp-4.2.5/dhcpd.conf.example复制到/etc/dhcp/dhcpd.conf
默认以分配的地址信息在/var/lib/dhcp/dhcpd-leases

全局配置：

default-lease-time  //默认租期时间
max-lease-time      //最大租期时间

子网配置：

subnet IPADDR netmask NETMASK { //定义子网
    range IPRANG;    //定义IP的范围
    option route IP;    //定义默认网关
    option domain-name-servers  DNS;//定义默认DNS
    filename        //定义引导文件名称
        示例：
            filename "pxelinux.0";
    next-server     //提供引导文件的服务器的IP地址
        示例：
            next-server 192.168.50.7;
}

单个主机配置：

host 名称 {
    hardware ethernet   //主机的MAC地址
    fixed-address       //固定分配一个IP地址
}

重新获取地址：dhclient -d，如出现bound new ip表示dhcp服务器正常启动

TFTP：

yum install tftp-server，服务器端
yum install tftp，客户端

systemctl start tftp.service
默认监听 69/udp

默认路径 /var/lib/tftpboot
PXE，Preboot Eccution Environment(预启动执行环境)，网络自动安装单个版本系统
yum install syslinux(centos7)，syslinux-nonlinux(centos6)

mkdir /var/lib/tftpboot/pxelinux.cfg/

cp /usr/share/syslinux/{pxelinux.0,menu.c32} /var/lib/tftpboot

cp /media/cdrom/isolinux/{initrd.img,vmlinuz} /var/lib/tftpboot/

cp /media/cdrom/isolinux/isolinux.cfg /var/lib/tftpboot/pxelinux.cfg/default

使用网络启动主机，实现网络自动安装

Cobbler

cobbler基本用法

cobbler安装配置：

机器需连互联网

1.yum install cobbler dhcp

2.确认httpd、tftp-sever已经启动且已经开机自启systemctl start httpd tftpd

3.启动cobbler，systemctl start cobblerd.service

4.初次cobbler check #检查cobbler运行环境

解决报错

修改/etc/cobbler/setting

6.systemctl restart cobblerd

7.执行命令下载所需文件

  • cobbler get-loaders

8.再次cobbler check检查cobbler运行环境（dhcp启动失败）

修改cobbler的dhcp模板文件

vim /etc/cobbler/dhcp.template

10.同步配置到数据目录

  • cobbler sync

应答文件

可自己准备，也可命令生成

命令生成

指定光盘文件，导入操作系统，并会自动生成应答文件：生成centos6和centos7，需挂载6和7系统光盘

生成后cobbler会自动生成最小化安装的应答文件

手动准备

11.把已准备好的应答文件拷贝至/var/lib/cobbler/kickstarts目录下

cp ks{6,7}.cfg /var/lib/cobbler/kickstarts

使应答文件添加到对应的启动菜单项

cobbler web的实现

cobbler web安装

  1. yum install cobbler-web

    • 安装后会生成httpd服务的子配置文件，需重启httpd. 默认监听443端口加密协议

  2. systemctl reload httpd

  3. 访问浏览器https://192.168.50.7/cobbler_web

    • 默认用户名和密码都为cobbler

  • 使用htdigest命令创建新用户

    • htdigest /etc/cobbler/users.digest Cobbler wpy #生成wpy用户

修改cobbler-web的验证方式

默认使用验证方法：module = authn_configfile

可修改为pam模块验证,使用系统用户作为管理

  • 创建用户：useradd -s /bin/nologin cobbleradmin
  • 设置用户密码：
    • passwd cobbleradmin
