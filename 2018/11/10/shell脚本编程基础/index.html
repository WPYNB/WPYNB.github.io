shell脚本编程基础

发表于 2018-11-10 | 分类于 linux ， shell脚本编程基础

什么是shell

它是一种程序设计语言。作为命令语言，它交互式解释和执行用户输入的命令或者自动地解释和执行预先设定好的一连串的命令；作为程序设计语言，它定义了各种变量和参数，并提供了许多在高级语言中才具有的控制结构，包括循环和分支。

shell脚本的用途有：
自动化常用命令
执行系统管理和故障排除
创建简单的应用程序
处理文本或文件

创建shell脚本 第一行必须包括shell声明序列：#!/bin/bash(首行shebang机制)

编写shell脚本

用文本编辑器创建一个文件第一行写#!/bin/bash(首行shebang机制)声明
想要给命令添加注释可以开头加#号，脚本不会运行#开头的行

脚本规范：

脚本代码开头约定,以下都是可选项

1、第一行一般为调用使用的语言
2、程序名，避免更改文件名为无法找到正确的文件
3、版本号
4、更改后的时间
5、作者相关信息
6、该程序的作用，及注意事项
7、最后是各版本的更新简要说明

变量

变量，是指没有固定的值，可以改变的数。
比如说一个a=1 其实程序并不把a当成数字1，他只会当成一个字符串，除非给他强制转换 a=int(1)才会把a当成一个数字1. linux中bash默认会自动做转换。

变量命名法则：

1、不能使程序中的保留字：例如if, for
2、只能使用数字、字母及下划线，且不能以数字开头
3、见名知义
4、统一命名规则：驼峰命名法

变量引用：${name} $name

“ “：弱引用，其中的变量引用会被替换为变量值
‘ ‘：强引用，其中的变量引用不会被替换为变量值，而保持原字符串
定义变量前面要加$符号

删除变量

显示已定义的所有变量：set
删除变量：unset name

环境变量

在linux中我们常用的变量叫普通变量，我们定义一个变量之后只能在当前的bash执行，如果新开一个bash就无法执行在上一个bash里建立的变量。

位置变量

位置变量是在执行一个程序的时候后面跟上内容（可以是命令），它会写到程序里预留的$的位置
$1, $2, …：对应第1、第2等参数
$0: 命令本身
$: 传递给脚本的所有参数，全部参数合为一个字符串
$@: 传递给脚本的所有参数，每个参数为独立字符串
$@ $ 只在被双引号包起来的时候才会有差异

只读变量（常亮）

只读变量创建了之后只能查看不能修改，也不能删除,下面两个命令创建效果一样
查看只读变量：
readonly–p

查看直接结果退出状态

如果想知道命令执行的成功与否可用命令来查看，返回0即为成功，返回1-255其中的数字就为失败
但是只能显示最后一条命令的成功或者失败，也可以自己指定状态码：exit number

退出状态码

bash自定义退出状态码
exit [n]：自定义退出状态码
注意：脚本中一旦遇到exit命令，脚本会立即终止；终止退出状态取决于exit命令后面的数字
注意：如果未给脚本指定退出状态码，整个脚本的退出状态码取决于脚本中执行的最后一条命令的状态码

随机数生成器

bash有内建的随机数生成器：$RANDOM（0-32767）
例：做一个脚本，内容要有颜色，并且颜色随机

逻辑运算

逻辑运算主要体现在真和假
0为true
1位false
与运算:必须都是true否则都是false
1 与 1 = 1
1 与 0 = 1
0 与 1 = 1
0 与 0 = 0
或运算：只要其中一个为true，就是true ，否则就是false
1 或1 = 1
1 或0 = 0
0 或1 = 0
0 或0 = 0
非：取反，真为假，假为真
! 1 = 0 ! true
! 0 = 1 ! false
异或：^ 异或的两个值,相同为假，不同为真

短路与短路或

短路与（& & ），如果前面的项有结果又为假的，那么结果必定为假，如果前面的项为真，那么后面的必须执行
第一个为1，结果必定为1
第一个为0，第二个必须要参与运算
短路或（||）如果前面的为真，那么就一定为真，且后面的一定不会再执行，如果前面的为假，那么后面的必须执行
第一个为0，结果必定为0
第一个为1，第二个必须要参与运算

条件测试

判断某需求是否满足，需要由测试机制来实现test,被判断的字符最好用” “ 引起来，不管是数字还是字符串都可以测试。

也可以测试一个单独的字符串是否为空 -z 如果字符串为空，就判断为真 -n 如果字符串不为空就为真，为空就是假

bash的数值测试:

-gt是否大于
-ge是否大于等于
-eq是否等于
-ne是否不等于
-lt是否小于
-le是否小于等于
=~ 左侧字符串是否能够被右侧的PATTERN所匹配

条件性的执行操作符

我们可以用短路与和短路或来实现命令是否继续执行
例：判断一个用户是否存在，如果存在则不做操作，如果不存在则创建这个用户。

取出硬盘利用率大于10就提示警告

ping 一个地址，如果能通就显示up,不同就显示down

Bash的文件测试:

-r FILE：是否存在且可读
-w FILE: 是否存在且可写
-x FILE: 是否存在且可执行

文件特殊权限测试：

-u FILE：是否存在且拥有suid权限
-g FILE：是否存在且拥有sgid权限
-k FILE：是否存在且拥有sticky权限

存在性测试:

-a FILE：同-e
-e FILE: 文件存在性测试，存在为真，否则为假

存在性及类别测试:

-b FILE：是否存在且为块设备文件
-c FILE：是否存在且为字符设备文件
-d FILE：是否存在且为目录文件
-f FILE：是否存在且为普通文件
-h FILE 或-L FILE：存在且为符号链接文件
-p FILE：是否存在且为命名管道文件
-S FILE：是否存在且为套接字文件

文件大小测试：

-s FILE: 是否存在且非空

文件是否打开：

-t fd: fd文件描述符是否在某终端已经打开
-N FILE：文件自从上一次被读取之后是否被修改过
-O FILE：当前有效用户是否为文件属主
-G FILE：当前有效用户是否为文件属组

双目测试：

FILE1 -efFILE2: FILE1是否是FILE2的硬链接
FILE1 -ntFILE2: FILE1是否新于FILE2（mtime）
FILE1 -otFILE2: FILE1是否旧于FILE2
