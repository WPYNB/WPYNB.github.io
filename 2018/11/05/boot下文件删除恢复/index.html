CentOS6 boot下文件删除恢复

发表于 2018-11-05 | 分类于 linux ， boot下文件删除恢复

删除/boot下initramfs-2.6.32-754.el6.x86_64.img恢复

1.光盘进入救援模式

进入引导界面，选择Rescueinstalled system，Choose a language，选择English，Keyboad Type选择us，Rescue method选择Local CD/DVD，Setup networking选择no，Rescue选择Contiune，Rescue选择ok，下一个界面再次选择ok，shell start shell，启动shell。

删除/boot下vmlinuz-3.10.0-862.el7.x86_64恢复

2.光盘进入救援模式

进入引导界面，选择Rescueinstalled system，Choose a language，选择English，Keyboad Type选择us，Rescue method选择Local CD/DVD，Setup networking选择no，Rescue选择Contiune，Rescue选择ok，下一个界面再次选择ok，shell start shell，启动shell。

3.删除/boot/grub/grub.conf文件恢复

1.重启机器，显示如下界面

2.写入以下内如回车启动

成功进入系统后在/boot/grub/重写一份grub.conf文件

删除/boot下所有文件恢复

1.光盘进入救援模式，挂载光盘，安装kernel-2.6.32-754.e16.x86.rpm。因为所在目录不是真实根目录，所以跟上–root=/mnt/sysimage (真实根目录)–force （强行安装）

2.切换到真实根目录，再恢复grub,手写一份grub.conf文件在grub目录下

退出重启即可
